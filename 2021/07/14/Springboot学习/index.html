一、SpringBoot属于Spring

Spring能干什么？

  • 微服务
  • 异步反应式开发
  • 云
  • 开发网络应用
  • 灵活部署
  • 业务监控
  • 自动化

二、 为什么使用Springboot？

能快速创建出生产级别的Spring应用

优点：

  • 创建独立Spring应用

  • 内嵌web服务器

  • 自动starter依赖，简化构建配置

  • 自动配置Spring以及第三方功能

  • 提供生产级别的监控、健康检查及外部化配置

  • 无代码生成、无需编写XML

SpringBoot是整合Spring技术栈的一站式框架

SpringBoot是简化Spring技术栈的快速开发脚手架

缺点：

  • 人称版本帝，迭代快，需要时刻关注变化
  • 封装太深，内部原理复杂，不容易精通

三、时代背景

3.1 微服务

  • 微服务是一种架构风格

  • 一个应用拆分为一组小型服务

  • 每个服务运行在自己的进程内，也就是可独立部署和升级

  • 服务之间使用轻量级HTTP交互

  • 服务围绕业务功能拆分

  • 可以由全自动部署机制独立部署

  • 去中心化，服务自治。服务可以使用不同的语言、不同的存储技术

3.2 分布式

SpringBoot + SpringCloud

3.3 云原生

Cloud Native
